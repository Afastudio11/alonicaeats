1. Bug Fix: Gagal Membuka Shift (Error 401: Unauthorized)

Error ini terjadi berulang kali dan mencegah Kasir memulai sesi kerja. Ini adalah masalah blocker.

A. Detail Kejadian

    Modul: Manajemen Shift.

    Langkah Reproduksi:

        Login sebagai user Kasir (username: kasir1).

        Akses menu Manajemen Shift.

        Input Kas Awal (misal: 30000).

        Tekan tombol "Buka Shift".

    Hasil yang Diharapkan: Shift berhasil dibuka, sistem menampilkan Toast sukses, dan user diarahkan ke Menu Order.

    Hasil yang Ditemukan (Error): Muncul banner berwarna merah di bawah dengan pesan: "Gagal Membuka Shift 401: Unauthorized".

B. Investigasi Teknis yang Diminta

Mohon untuk melakukan investigasi API dan backend secara mendalam:

    Cek API Endpoint: Identifikasi endpoint API yang dipanggil saat tombol "Buka Shift" ditekan (kemungkinan POST /api/shifts/open atau sejenisnya).

    Cek Header Otentikasi: Verifikasi apakah token otentikasi (misalnya, Bearer Token di Header Authorization) terkirim dengan benar saat API dipanggil. Pastikan token tersebut tidak hilang atau kadaluarsa di sisi klien (browser/aplikasi).

    Cek Server-side Log: Periksa log di sisi backend saat request 401 terjadi.

        Apakah request mencapai server?

        Jika ya, mengapa server menganggap token tersebut Unauthorized (tidak valid, expired, atau scope/role tidak diizinkan)?

    Cek Role Access Control: Konfirmasi bahwa role Kasir secara definitif memiliki izin yang benar (di dalam middleware otorisasi) untuk menjalankan aksi shifts/open.

2. Bug Fix: Gap Kosong / Area Putih Besar di Dashboard Kasir

Area gap kosong ini menunjukkan ketidakmampuan layout untuk mengisi konten atau komponen utama gagal di-render.

A. Detail Kejadian

    Lokasi: Halaman Dashboard Kasir (Halaman Home setelah Login Kasir, lihat gambar image_7fea05.png).

    Masalah: Seluruh area kanan layar (area konten utama) didominasi oleh ruang kosong berwarna putih, di atas baris Summary Cards (Total Orders, Completed Orders, dll.).

B. Investigasi Teknis yang Diminta

Mohon untuk fokus pada rendering komponen di sisi frontend:

    Cek Component Rendering:

        Komponen atau widget apa yang seharusnya mengisi ruang kosong tersebut (misalnya: Daftar Pesanan Aktif, Notifikasi Dapur, atau Grafik Harian)?

        Pastikan komponen tersebut tidak disembunyikan (hidden via CSS: display: none; atau visibility: hidden;) atau diatur ukurannya menjadi nol.

    Cek Kondisional Rendering: Jika area tersebut diisi secara kondisional (misalnya, hanya muncul jika ada data), pastikan:

        Data sudah dimuat dan tersedia.

        Tidak ada bug di logic yang menentukan kapan komponen tersebut harus di-render (if (data) { render component }).

        Jika data memang kosong (misal: belum ada pesanan), tampilkan placeholder yang rapi, bukan hanya ruang kosong.

    Cek CSS/Grid Layout:

        Periksa apakah ada pengaturan CSS Grid atau Flexbox yang salah yang menyebabkan komponen anak berukuran kecil atau terdorong keluar dari pandangan (overflow atau margin/ padding yang terlalu besar) sehingga menyisakan ruang kosong besar.
