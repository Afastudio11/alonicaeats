Prompt Pengembangan Fitur Dashboard Self-Order

Kami membutuhkan pengembangan dan penyesuaian fitur pada dashboard admin dan kasir.

1. Dashboard Kasir

A. Manajemen Shift & Laporan Harian (Closingan)

    Buka/Tutup Shift: Kasir harus memulai sesi kerja dengan "Buka Shift" dan mengakhirinya dengan "Tutup Shift". Setiap shift harus memiliki ID Sesi Unik.

    Kas Masuk/Keluar: Fitur untuk mencatat setoran awal (kas masuk) dan Pengeluaran Dadakan (kas keluar) yang tidak terkait dengan transaksi penjualan.

    Rekap Akhir Shift (Closingan):

        Sistem menampilkan total pendapatan tercatat untuk shift tersebut.

        Kasir menginput jumlah uang fisik yang dihitung.

        Sistem menampilkan dan mencatat adanya Selisih (Kelebihan/Kekurangan Dana) pada laporan sesi.

B. Refund / Void

    Pembatalan Transaksi: Kasir dapat mengajukan Void (pembatalan) atau Refund (pengembalian dana).

    Otorisasi Admin/Owner: Proses Refund/Void wajib memerlukan input password atau approval dari akun Admin/Owner sebelum dieksekusi, dan riwayatnya harus tercatat di log Admin.

2. Dashboard Admin

A. Manajemen Akun Kasir

    Fungsi Dasar: Admin dapat menambah, mengedit, atau menonaktifkan akun kasir.

    Pengaturan Akun: Fitur untuk mengganti ID Pengguna (username) dan Password kasir yang ada.

B. Laporan dan Audit Kasir (Laporan per Shift)

    Audit Laporan per Sesi: Admin harus dapat melihat semua laporan kasir berdasarkan ID Sesi/Shift yang unik.

    Detail Laporan: Laporan harus mencakup Nama Kasir, Waktu Mulai/Selesai Shift, Detail Closingan, dan semua Pengeluaran Dadakan yang dicatat.

    Pelacakan Riwayat: Sediakan filter tanggal untuk menganalisis laporan kasir semua sesi/shift minimal 1 bulan terakhir.

C. Halaman Analisis dengan Ekspor Laporan

    Fungsi Analisis: Buatkan halaman terpisah untuk analisis penjualan.

    Ekspor Rangkuman Data: Ketika tombol download (ekspor) ditekan, file yang dihasilkan (CSV/Excel) HANYA boleh berisi rangkuman data dari dashboard analisis tersebut, bukan data transaksi per baris yang detail.

D. Perubahan Desain Tampilan Kategori Menu

    Kategori-kategori menu harus ditampilkan secara berjejer ke samping (datar), bukan berjejer turun.

    Layout kategori harus dapat digeser secara horizontal (scrollable) agar tampilan tetap rapi di berbagai ukuran layar.
